# Data Protection and Privacy

## Objective
Understand how encryption, backups, secure deletion, account protection, and privacy settings help protect sensitive information and reduce cybersecurity risks.
---

# Encryption

## Overview
Encryption converts readable data into unreadable ciphertext to prevent unauthorized access.

## Security Benefits
- protects confidential information
- reduces unauthorized access risks
- secures data during storage and transmission

## Common Use Cases
- file encryption
- full disk encryption
- secure communications
- VPN protection

## Risks Without Encryption
- data exposure
- credential theft
- privacy violations
- financial loss
---

# Encrypting File System (EFS)

## Overview
Encrypting File System (EFS) is a Windows feature used to encrypt files and folders.

## Security Benefits
- protects sensitive files
- restricts unauthorized access
- integrates with Windows user accounts

## Limitations
- unavailable on some Windows editions
- encrypted files may become inaccessible if recovery keys are lost
---

# Data Backup Security

## Importance of Backups
Regular backups help organizations recover from:
- ransomware attacks
- accidental deletion
- hardware failures
- system corruption

## Backup Best Practices
- maintain regular backups
- store backups securely
- use offline or cloud backups
- test backup restoration procedures
---

# Secure Data Deletion

## Overview
Deleted files may still remain recoverable from storage devices.

## Secure Deletion Methods
- overwrite data multiple times
- use secure deletion tools
- physically destroy storage devices when necessary

## Security Risks
Improper deletion may lead to:
- data leakage
- unauthorized recovery
- compliance violations
---

# Password and Account Security

## Best Practices
- use strong passphrases
- enable multi-factor authentication
- avoid password reuse
- secure cloud accounts

## Common Threats
- brute-force attacks
- credential theft
- phishing attacks
- unauthorized access
---

# Privacy and Terms of Service

## Key Considerations
Before using online services:
- review Terms of Service
- understand data usage policies
- verify privacy settings
- review provider security policies

## Privacy Risks
- excessive data sharing
- third-party access
- public exposure of personal information
---

# Public and Cloud Data Risks

## Risks
Data stored online may be:
- exposed through misconfigurations
- accessed by attackers
- shared beyond intended audiences

## Security Measures
- configure privacy settings
- restrict file sharing
- monitor account activity
- encrypt sensitive uploads
---

# SOC Relevance

SOC analysts monitor unauthorized access attempts, suspicious account activity, abnormal file access, data exfiltration attempts, and security alerts related to encryption, backups, and account compromise.
---

# Key Learning

Strong encryption, secure backups, privacy awareness, proper account security, and secure data handling significantly improve organizational cybersecurity resilience.
