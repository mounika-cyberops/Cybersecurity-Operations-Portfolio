# Data States in Cybersecurity

## Objective
Understand how data exists and moves within information systems and why each state requires security protection.

## Data in Process
Data in process refers to data actively being used or modified by applications or systems.

Examples:
- database updates
- user authentication
- application processing

## Data at Rest
Data at rest refers to stored data located on storage devices or databases.

Examples:
- hard drives
- cloud storage
- backup files
- databases

## Data in Transit
Data in transit refers to data moving between systems or networks.

Examples:
- emails
- website traffic
- file transfers
- API communication

## Cybersecurity Relevance
Security teams protect data in all states to reduce risks such as:
- unauthorized access
- data theft
- interception
- tampering

## Key Learning
Effective cybersecurity requires protecting data while it is being processed, stored, and transmitted across systems and networks.
